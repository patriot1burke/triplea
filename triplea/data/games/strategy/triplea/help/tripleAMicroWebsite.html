Posts to a custom TripleA game website

This poster will send your savegame to a website, where it can be accessed by you and your friends. The website will send out email notifications to whoever is the next player, that their turn is ready to start. The website may also show various statistics about the game in progress.

How to Use

Starting new games:
  1. The player which comes first (e.g. Russians) should set up a new game by setting the web site URL (e.g. 'http://triplea.myserver.com/') and must choose a game name. NOTE: Unicode characters (Umlauts, etc.) are not being handled correctly, yet!
  2. Click 'Initialize' and assign the players.
  3. After successful initialization click 'Play'.
  4. Make your move and post the game state at the end of your turn.

Making your turn:
  1. The web site may send you an email, whenever it's your turn (if 'Send Mails' is checked)
  2. When it's your turn, download the latest save game from the TripleA game web site, load into TripleA, play and finally post at the end of your turn.

Managing your games:
  1. There is currently not yet any way to set a game as won or lost, or to surrender a game.
  2. There is currently not yet any way for users to delete finished games.


How to Create custom TripleA game website

This poster is built for managing TripleA games with a private or public website.
Requirements:
  1. A public internet domain or dynamic DNS.
  2. A small webserver which supports PHP scripts and optionally is able to send mails via PHP's mail() function.

Installation:
  1. The public (sub-)domain or the dynamic DNS should point to your webserver.
  2. Get and extract the PHP scripts for your private TripleA game web site from X?X!?.
  3. Modify the settings.php file in order to adjust the list of human players.
  4. Upload the scripts to your webserver.

Managing your players and games:
  1. The set of available players is managed by the web site admin through the 'settings.php'.
  2. The web site administrator needs to delete the according files in the 'games' folder at the server.
